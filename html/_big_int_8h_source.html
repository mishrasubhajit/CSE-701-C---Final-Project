22  * Author: Nedim Srndic

25  * A class representing a positive or negative integer that may
26  * be too large to fit in any of the standard C++ integer types
27  * (i. e. 2^128 is "just" 39 digits long).
28  * The digits are stored in a dinamic array of tipe unsigned char*,
29  * with values from 0 to 9 (not '0' to '9'), so that the CPU can
30  * add/subtract individual digits.
31  *
32  * The array has "length" memory locations, one byte each (the size of
33  * unsigned char is probably one byte). There are "digitCount" digits actually
34  * in use, the rest is spare space.
35  * The number of digits is constrained by available memory and the limit of the
36  * unsigned long long int type used for indexing (the "length" property).
37  * The individual digits are stored right-to-left, to speed up computing and
38  * allow for faster growth of numbers (no need to reallocate memory when
39  * the digitCount grows).
40  *
41  * The class handles its own memory management. There are no memory leaks
42  * reported until this date.
43  * When creating a BigInt from const char* or unsigned long long int,
44  * copying from an other BigInt with (digitCount + 2 <= length)
45  * (soon to be full), new memory is allocated and
46  * length is adjusted to (length * FACTOR + 1). This is done to expand the
47  * capacity of the digits array to accomodate potential new digits.
48  * When assigning a BigInt "bInt" that is twice as small or bigger than *this,
49  * the length is set to (bInt.length + 2).
50  *
51  * BigInt supports:
52  *
53  * - addition (unary +, binary +, +=, prefix ++, postfix ++)
54  *
55  * - subtraction (unary -, binary -, -=, prefix --, postfix --)
56  *
57  * - multiplication (*, *=)
58  * For multiplication, one can choose between the Square and multiply
59  * or Karatsuba algorithm, or long multiplication at compile time
60  * (this can be done by defining or undefining the macro "KARATSUBA"
61  * in BigInt.cpp).
62  * The Karatsuba algorithm multiplies integers in O(n^log2(3))
63  * complexity. log2(3) is approximately 1.585, so this should be
64  * significantly faster than long multiplication, if the numbers are
65  * big enough. Currently, the long multiplication is better implemented,
66  * and runs faster than the Karatsuba multiplication for numbers shorter
67  * than about 100 digits.
68  *
69  * - C-style integer division (/, /=)
70  *
71  * - C-style integer division remainder (%, %=)
72  * When calculating the remainder, the number is first divided.
73  *
74  * - comparison (==, !=, <, <=, >, >=)
75  * All of the <, <=, >, >= operators are equally fast.
76  *
77  * - exponentiation (GetPower(), SetPower(), GetPowerMod(), SetPowerMod())
78  * For exponentiation, the Exponantiation by squaring
79  * (or Square and multiply or Binary exponentiation) algorithm is used.
80  * It uses O(log(n)) multiplications and therefore is significantly faster
81  * than multiplying x with itself n-1 times.
82  *
83  * In addition to mathematical operations, BigInt supports:
84  *
85  * - automatic conversion from const char *, std::string and unsigned long long int
86  * - safe construction, copying, assignment and destruction
87  * - automatic conversion to std::string
88  * - writing to the standard output (operator <<(std::ostream, BigInt))
89  * - reading from the standard input (operator >>(std::istream, BigInt))
90  * - getting and setting individual digits (GetDigit(), SetDigit())
91  * - returning the number of digits (Length())
92  * - returning a string of digits (ToString())
93  * This can be useful for human-readable output.
94  * - returning a value indicating wether the number is odd (IsOdd())
95  * - returning a value indicating wether the number is positive (IsPositive())
96  * - returning a value indicating wether the BigInt equals zero (EqualsZero())
97  * The fastest way to determine this.
98  * - returning absolute value (Abs())
99  *
100  * There are a few static constants defined in this file:
101  *
102  * - BigIntZero : a zero of type BigInt
103  * If you need a zero fast, use this.
104  * - BigIntOne : a one of type BigInt
105  * If you need a one fast, use this.
106  *
107  * ****************************************************************************
108 */
